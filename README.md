# HTML5 Semantic Structure & Accessibility Portfolio Website

## Project Description
This project is a multi-page personal portfolio website developed using Semantic HTML5 elements and WCAG accessibility guidelines. The website demonstrates proper document structure, accessibility best practices, and SEO optimization techniques.

## Features
- Semantic HTML5 tags (`header`, `nav`, `main`, `section`, `article`, `footer`)
- ARIA labels and roles for improved screen reader support
- SEO-friendly meta tags
- Accessible contact form with proper labels
- Keyboard and tab navigation support
- Responsive viewport configuration
- Structured heading hierarchy

## Pages
### Home Page (index.html)
- Introduction section
- Skills section

### About Page (about.html)
- Personal information
- Education details

### Contact Page (contact.html)
- Accessible contact form
- Name, Email, and Message fields

## Accessibility Features
- ARIA attributes for navigation and forms
- Proper heading structure (H1 → H2 → H3)
- Screen-reader-friendly content
- Keyboard-accessible navigation
- Form validation using required fields

## SEO Features
- Descriptive page titles
- Meta description tags
- Keywords meta tags
- Author information
- Robots indexing tag

## Technologies Used
- HTML5
- Semantic Web Standards
- WCAG Accessibility Guidelines
- ARIA
- SEO Best Practices
